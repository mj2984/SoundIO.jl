# SoundIO.jl Architecture Overview

SoundIO.jl is structured to preserve libsoundio’s low-level performance while providing a Julian interface built on type specialization, multiple dispatch, and safe resource management.

---

## Module Layout

### `SoundIO.jl`
Entry point of the package.  
Exports public types and functions, initializes constants, and defines high-level helpers such as `play_audio`.

### `SoundIOConstants.jl`
Defines:
- error codes  
- channel layouts  
- sample formats  
- backend identifiers  

All constants are mapped to idiomatic Julia symbols.

### `SoundIOStructs.jl`
Typed wrappers around libsoundio C structs:
- `SoundIOContext`
- `SoundIODevice`
- `SoundIOInStream`
- `SoundIOOutStream`

Each wrapper stores:
- the raw C pointer  
- metadata extracted from libsoundio  
- Julia-side state (callbacks, buffers, flags)

These wrappers enforce type safety while still allowing pointer access when needed.

### `SoundIOMethods.jl`
Implements the functional API:
- connecting/disconnecting contexts  
- enumerating devices  
- opening/closing streams  
- reading/writing buffers  
- generating callbacks  
- synchronization (`wait(context)` / `wait(device)`)

This file contains the core logic binding Julia to libsoundio.

### `SoundIOPrettyPrintMethods.jl`
Custom `show` overloads for:
- contexts  
- devices  
- streams  

These provide rich REPL inspection and self-documenting behavior.

## Callback Architecture

Callbacks are generated by specializing on the concrete stream type.

~~~julia
cb = make_sound_output_callback(stream, userfunc)
stream.write_callback = cb
~~~

The generated callback:
- is compiled specifically for that stream  
- captures arbitrary Julia state  
- exposes safe Julia arrays or raw pointers  
- avoids dynamic dispatch  
- integrates directly into the stream object  

This design gives C-level performance with Julia-level clarity.

---

## Synchronization Model

SoundIO.jl exposes both libsoundio synchronization strategies.

### Fast Wait
~~~julia
wait(context)
wait(device)
~~~

Used for:
- real-time synthesis  
- pinned-thread workers  
- low-latency loops  

### Event-Driven Wait
Callbacks trigger Julia tasks at lower frequency.

Useful for:
- GUIs  
- monitoring  
- asynchronous pipelines  

---

## Resource Management

SoundIO.jl overloads:
- `open(device)`  
- `close(stream)`  
- `isopen(stream)`  
- `show`  

This makes audio devices behave like native Julia I/O resources.

---

## Extensibility

The architecture is intentionally minimal, enabling higher-level tools such as:
- lock-free ring buffers  
- DSP pipelines  
- audio graphs  
- async processors  
- custom callback generators  

Multiple dispatch makes extension natural and non-intrusive.
